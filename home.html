Automation

Let the robots help you

How long did your last release take? More than 30 minutes? More than an hour? All day?! Days!!!??? Ok, no problem. Did you find any bugs afterward? How many?!?! IT'S TIME FOR AUTOMATION! Look, if you've had to get code into a production environment, even once, you know that it can be a complex process. If not, then your team is probably automating deployment and testing.

Build and Deployment Automation

Start by making your build and deploy process push button easy. There are loads of CI products that can make this kind of thing very easy: Jenkins, Hudson, Bamboo, Team City, and on and on and on. Make the complicated process of compiling, building, and deploying really easy by having a CI server do it for you the same way every time. Repeatable. Reliable. Beautiful.

Test Automation

Ok, so know that your code is getting shipped effortlessly let's make sure everything is solid as a rock by testing it out. There are several different ways we can cover our product with tests. We can test individual pieces of code with unit tests. We can cover entire modules with integration tests. Often times automating the browser to simulate real usage is valuable so you might want to protect your product with browser automation. We can also protect against high volume by load testing.

In all cases. Test are automated and repeatable with every deployment. All you have to do is click the button.
